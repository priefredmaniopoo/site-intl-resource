# site-intl-resource

## 简介

`sit-intl-resource` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为通用资源存储与展示空间，用于存放各类静态 HTML 内容。

## 目录说明

仓库根目录下存放各独立 HTML 文件及其相关资源。典型结构如下：

```
site-intl-resource/
├── index.html          # 项目首页（可选）
├── page-example.html   # 示例页面
├── assets/             # 共用资源文件夹
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `index.html`：可选的项目导航或说明页面。
- `*.html`：各独立 HTML 页面，文件命名应反映内容用途。
- `assets/`：存放页面共用的样式、脚本、图片等资源。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档文件，彼此之间无功能依赖。每个页面可单独通过 GitHub Pages 或其他静态托管服务访问。

### 归档原则

- 页面内容保持稳定，不依赖外部动态服务。
- 资源文件尽量内联或放置在 `assets/` 目录下。
- 不包含任何跟踪、广告或外部请求（除非必要）。

## 维护说明

### 添加新页面

1. 在根目录下创建新的 `.html` 文件。
2. 如需共用资源，请放入 `assets/` 对应子目录。
3. 确保页面内部链接使用相对路径。

### 更新已有页面

- 直接修改对应 HTML 文件，保持向后兼容。
- 若涉及资源文件变更，请同步更新 `assets/` 目录。

### 删除页面

- 移除对应 `.html` 文件及相关资源引用。
- 如有必要，更新 `index.html` 中的导航链接。

## 贡献

- 新增页面不违反任何法律法规或平台政策。

- 不引入恶意代码或外部依赖。
